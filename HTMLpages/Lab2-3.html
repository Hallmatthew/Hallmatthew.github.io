1. Define the problem This step involves gathering the requirements of the program needed to solve the problem.Questions that should be answered
  2. What data do we need to proccess?
  3. How should we process the data?
  4. What ouput should the program deliver?
  5. How should the user interface look?
  6. Design the Program Programs design focuses on breaking down the program into smaller, more mangageble components.Various tools are used to help design a program. Commonly used tools:
  7. Structure Charts
  8. Algorithm
  9. Flow Chart
  10. Decision tables
  11. Pseduocode
  12. Code the program Once the program has been designed, a developer then writes the code. Codeing often involves translating the design from charts and pseducocode into a specific programming language.Usually the language is chosen during the "defining" stage.
  13. Testing and Debugging After code has been written this stage revolves around fixing any bugs in the program.There are two kinds of errors that cause bugs.
  14. a.Syntax Errors: These are like grammatical mistakes in the code.
  15. b.Logical Error:These mistakes might cause an other than intended output but the code still works.
    16. The programmer must fix these mistakes by carefully exammining the already written code for errors.
  16. Documentation After coding has been completed, the programmer must write a mannual that generally contains the following:
  17. An overview of the programs functionality.
  18. In-Depth explanations of program features.
  19. Documentation of all commands.
  20. A thorough description of errors messages generated by the program.
  21. Deploying and Maintaining the Program A software team will be responsible for fixing new errors and updating the software with new features. When a new feature should be added to the program, the cycle restarts as developers begin to define what problem the new fature will solve.
